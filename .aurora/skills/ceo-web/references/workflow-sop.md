# Workflow SOP: A Ordem das Operações (Sam Crawford Protocol)

Este documento define o processo rígido de construção de qualquer nova presença digital no ecossistema Aurora. 

## 1. O Funil de Construção (Gate Obrigatório)

Nenhuma página inicia no código ou no design sem atravessar este funil:

1. **Estratégia (The 3 Questions)**:
   - Quem é o visitante?
   - O que eles precisam saber para agir?
   - Qual a ÚNICA ação desejada?
   
2. **Sitemap Direcional**:
   - Mapear a jornada lógica.
   - Definir ramificações se houver múltiplos ICPs.

3. **Wireframe de Texto (Hierarquia "Seca")**:
   - Definir seções, títulos e CTAs em texto puro/caixas cinzas.
   - Validar se a Prova Social está posicionada ANTES do CTA.

4. **Design Visual (A Estética de Serviço)**:
   - Aplicar Tipografia (Mood) e Cores (Jobs).
   - Aplicar White Space Disciplinado.

---

## 2. Regras de Hierarquia Visual

- **White Space**: Não é vazio; é o que foca a atenção. Se uma seção parece "apertada", ela está falhando na hierarquia.
- **Color Job**: O site deve ter no máximo 3 cores. Uma delas (a de maior contraste) deve ser usada **exclusivamente** para o CTA.
- **Tipografia**: Máximo de 2 fontes. Uma para headers (Headline) e uma para leitura densa (Body).

---

## 3. Pós-Lançamento: Launch is the Beginning

- SEO técnico deve ser estrutural (Tags semânticas, IDs únicos).
- Analytics deve mapear o "Monotrilho" para detectar desistências.
- Autonomia: O sistema deve permitir mudança de Copy sem dependência de engenharia.
