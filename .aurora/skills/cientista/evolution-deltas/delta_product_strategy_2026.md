# Evolution Delta: Aurora Product Strategy (Martell Protocol)

**Status:** ACTIVE
**Source:** CEO-Product Owner Ingestão Inicial (v1.0)
**Agent:** Cientista

## 1. Ontologia da Dor como Filtro de PM-Fit

A partir desta data, a Aurora Factory adota a **Ontologia da Dor** como o critério supremo de priorização:
- Se o fundador/desenvolvedor consegue descrever a dor do ICP melhor que ele mesmo, a feature tem sinal verde para o backlog.
- Features baseadas em "visão" ou "curiosidade" que não ataquem uma fratura exposta do mercado devem ser podadas (`feature pruning`).

## 2. Framework PSL (Point, Story, Lesson)

O **CEO-Product Owner** deve utilizar o PSL para orientar a distribuição do produto:
- **Ponto:** Qual é a dor específica?
- **Story:** Como o produto resolveu isso para alguém (ou para a própria Aurora)?
- **Lesson:** Qual a lição funcional/ROI que o usuário extrai?

## 3. Teoria das Restrições (TOC) em Distribuição

O gargalo de crescimento da Aurora foi identificado como o **atrito no onboarding**.
- O produto deve mover o usuário para a "Buying Zone" rapidamente.
- Uso de IA para fechamento direto (Direct Response Product) e diagnóstico clínico em vez de funis complexos.

## 4. Diretrizes de Engenharia para a Fábrica
- **Link Direto e Menos Telas:** Priorizar integrações transacionais diretas (ex: links de pagamento no chat).
- **Mapeamento de Pressão:** Inserir triggers nas interfaces onde o valor é percebido (ex: após um download bem-sucedido de documento processado).
