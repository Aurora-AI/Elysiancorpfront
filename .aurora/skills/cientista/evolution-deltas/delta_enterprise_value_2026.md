# Evolution Delta: Enterprise Value & Retention Architecture (Martell Protocol)

**Status:** ACTIVE
**Source:** CEO-Product Owner Ingestao: Enterprise Value
**Agent:** Cientista

## 1. Métrica Suprema: Time-to-First-Value (TTFV)

A Aurora Factory prioriza o **TTFV** como a métrica de sobrevivência:
- O Onboarding deve entregar um "Quick Win" em tempo recorde (idealmente < 1 hora de interação).
- A IA deve preencher lacunas de execução para que o usuário veja o output antes de configurar o sistema.
- **Ação:** Implementação de Dashboard de TTFV para monitoramento em tempo real.

## 2. Cancellation Capture Flow

Fica estabelecido o **Protocolo de Saída**:
- Nenhum cancelamento ocorre sem a captura sistemática do motivo.
- Os dados de churn devem ser injetados automaticamente na **Ontologia da Dor** para pautar a próxima sprint.

## 3. Gatilhos de Expansão (Upsell Contextual)

A interface deve mapear a jornada "3 minutos antes / 3 minutos depois":
- Codificar alertas contextuais baseados em sucesso.
- O upgrade deve ser uma consequência natural do aumento do uso e do valor percebido.

## 4. SOPs e Playbooks (Risco de Concentração)

A engenharia deve descentralizar o conhecimento:
- Uso do método *Camcorder* (SOPs via IA) para documentação automática de módulos.
- Garantir que o produto não dependa de conhecimento tácito individual (Enterprise Value Protection).
