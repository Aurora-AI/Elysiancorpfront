## PARTE I — FUNDAÇÃO FILOSÓFICA

### 1.1 O Que É a Elysian

Elysian é uma fábrica de produtos digitais AI-native que dá superpoderes a pessoas e profissionais.

Não substituímos humanos por IA.
Usamos IA para tornar humanos extraordinários.

A Elysian opera três verticais:

| Vertical | Produto | Promessa |
|---|---|---|
| Comercial | CRM Aurora + Silent Operator | Inteligência comercial que age |
| Jurídico | Elysian Lex | Precisão jurídica sem fricção |
| Criativo | Mad Lab Aurora | Experiências digitais impossíveis |

**A marca Elysian é a empresa. As verticais são os produtos.**
Nenhum produto domina a identidade da marca mãe.

### 1.2 A Origem do Nome

**Elysian** vem dos Campos Elísios — na mitologia grega, o destino dos heróis.
Não o paraíso dos mortos. O lugar onde os que viveram com propósito chegavam.

Essa referência não aparece como ornamento.
Ela aparece como **subtexto** — sentida, não explicada.

A Elysian não é o paraíso. É o caminho para o extraordinário.
Quem usa nossos produtos não se aposenta — se transforma.

### 1.3 A Missão em Uma Frase

> **Dar superpoderes a pessoas usando IA como infraestrutura invisível.**

O que isso significa na prática:
- A IA nunca é o produto. O humano elevado é o produto.
- A tecnologia é invisível. A transformação é visível.
- Confiança e empatia não são valores de parede — são requisitos de design.

### 1.4 O Princípio Da Vinci

A Elysian opera sob o princípio renascentista de beleza como verdade:

> *A beleza não é ornamento. É proporção. É o que acontece quando forma e função se tornam a mesma coisa.*

Isso significa:
- Nada é decorativo por acidente.
- Cada elemento tem razão de existir ou é removido.
- A beleza do produto é evidência da seriedade de quem o construiu.

### 1.5 A Lei dos Dois Mundos

A Elysian opera em dois mundos visuais com **igual autoridade**. Não escolhe entre eles.

```
MUNDO DARK    → Tensão. Máquina. Sistema. Dados. O peso do que existe.
               O preto que abafa. A dor que o cliente ainda não nomeou.
               Onde a tecnologia tem presença física.

MUNDO LUMINOSO → Paz. Humano. Proporção. Ar. O que se torna possível.
               O off-white que respira. A beleza como verdade.
               Onde a pessoa elevada existe.
```

**A jornada entre os dois mundos é a narrativa da marca.**
Dark → Luminoso = Máquina → Humano = Tensão → Paz = Potencial → Realizado.

Nenhum mundo é introdução do outro. Os dois *são* a Elysian.

## PARTE II — PERSONALIDADE E VOZ

### 2.1 Arquétipos da Marca

A Elysian opera em dois arquétipos simultâneos — não escolhe entre eles:

**O Sábio** — conhecimento profundo, sem arrogância.
Sabe mais do que fala. Quando fala, é preciso.

**O Herói** — age no mundo, transforma situações.
Não contempla. Constrói. Entrega. Eleva.

A tensão entre os dois é o que torna a marca interessante.
Um Sábio sem ação é acadêmico. Um Herói sem sabedoria é bruto.
A Elysian é os dois.

### 2.2 Tom de Voz

**O que a Elysian soa:**
- Direta sem ser fria
- Confiante sem ser arrogante
- Técnica sem ser hermética
- Provocadora sem ser agressiva

**O que a Elysian nunca soa:**
- Entusiasmada demais (startup hype)
- Corporativa e distante
- Condescendente com o cliente
- Apologética sobre tecnologia

### 2.3 Princípios de Escrita

**Frase curta antes da frase longa.**
Se uma ideia precisa de 40 palavras, ela começa com 6.

**Concreto antes de abstrato.**
"Reduz em 70% o tempo de resposta comercial" antes de "aumenta eficiência".

**O cliente é o herói. Sempre.**
A Elysian é o mentor. Nunca o protagonista.

**Dark Editorial na escrita:**
Quando o texto precisa criar tensão — quando estamos mostrando a dor que o cliente nem sabe que tem — a escrita vai para o silêncio. Frases fragmentadas. Pausa. Peso.

**Texto técnico como elemento visual:**
Dados, coordenadas, versões, identificadores de sistema (`AURORA_V8`, `SVRN_ID`, `ARTIFACT: 2026.Q2.CERT`) aparecem como elementos decorativos com intenção. Não como ruído — como evidência de que o sistema existe e funciona.

## PARTE III — SISTEMA TIPOGRÁFICO

### 3.1 Filosofia Tipográfica

A tipografia Elysian opera em **dois registros com igual autoridade**:

O Grotesco Humanista para quando a marca precisa ser próxima, confiante, acolhedora.
O espaço negativo e o silêncio para quando a marca precisa respirar e deixar o conteúdo falar.

No mundo dark: a tipografia é técnica, monospace em capslock, tracking extremo — evidência de sistema.
No mundo luminoso: a tipografia é o próprio conteúdo — massiva, preta, protagonista absoluta.

### 3.2 Fonte Principal — Display e Interface

**Família:** Söhne (Klim Type Foundry) — preferencial
**Alternativa de produção:** GT America (Grilli Type)
**Fallback web:** `system-ui, -apple-system, 'Helvetica Neue', sans-serif`

**Por que Söhne:**
Söhne carrega autoridade histórica com calor humanista. Não é neutra como Helvetica. Não é fria como Futura. É exatamente a tensão que a Elysian precisa: força que acolhe. No hero luminoso, em peso ExtraBold, ela tem presença física de objeto — não de fonte. O nome ELYSIAN em Söhne ExtraBold no off-white é a marca inteira em uma linha.

**Pesos em uso:**

| Peso | Uso | Mundo |
|---|---|---|
| Söhne Buch (Book 300) | Corpo de texto longo | Ambos |
| Söhne Leicht (Light 200) | Subtítulos de suporte, ar | Luminoso |
| Söhne Kräftig (Medium 500) | UI, labels, navegação | Ambos |
| Söhne Halbfett (SemiBold 600) | Headlines secundários | Luminoso |
| Söhne Dreiviertelfett (Bold 700) | Headline principal | Ambos |
| Söhne Extrafett (ExtraBold 800) | Display máximo, hero luminoso | Luminoso |

**Fonte técnica — Mundo Dark:**
Monospace em capslock, tracking extremo (+0.15em a +0.25em).
Uso: metadados de sistema, coordenadas, identificadores, labels técnicos.
`INDUSTRIAL INTELLIGENCE // AURORA_V8`
`ARTIFACT: 2026.Q2.CERT`
`SVRN_OPERATOR: ACTIVE`

**Fontes candidatas para o registro técnico:**
- JetBrains Mono (open source) — preferencial
- Space Mono (Google Fonts) — alternativa
- Söhne Mono (Klim) — se disponível na licença

### 3.3 Escala Tipográfica

```
Display XL:    80–200px / weight 800 / tracking -3px a -6px  ← Hero luminoso
Display L:     56–72px  / weight 700 / tracking -2px a -3px
Headline:      36–48px  / weight 600 / tracking -1px a -2px
Subheadline:   24–30px  / weight 500 / tracking -0.5px
Body L:        18–20px  / weight 300 / tracking 0    / line-height 1.7
Body:          15–16px  / weight 300 / tracking 0    / line-height 1.65
Technical:     11–13px  / weight 500 / tracking +0.15em a +0.25em / CAPSLOCK
Micro:         10–11px  / weight 500 / tracking +0.12em / metadata
```

**Regra de ouro:**
Display cresce → tracking vai para negativo (o nome ELYSIAN em 200px com tracking -6px tem peso de escultura).
Labels técnicos → tracking vai para positivo (legibilidade + evidência de sistema).
Nunca tracking positivo em display. Nunca tracking negativo em monospace técnico.

### 3.4 Hierarquia nos Dois Mundos

**Mundo Luminoso:**
- Display: Söhne ExtraBold 800, preto (#0A0A0A) sobre off-white (#F8F7F3)
- Corpo: Söhne Book 300, cinza escuro (#1A1A1A), 16px
- Labels: Söhne Medium 500 capslock, tracking extremo, muted

**Mundo Dark:**
- Texto técnico: Monospace capslock, branco 70–90% opacidade, tracking +0.2em
- Labels de vertical: `VERTICAL: LEX` / `STRATEGY: MAD LAB AURORA`
- Metadados: micro, opacidade 40–60% — presença sem peso
- O nome da marca no dark: Söhne SemiBold, branco, tamanho navegação (não display)

## PARTE IV — SISTEMA DE COR

### 4.1 Filosofia de Cor

A Elysian opera em dois mundos cromáticos com igual autoridade:

**No mundo luminoso:** a cor primária é ausência — o off-white que deixa o conteúdo respirar. Quando o mercado grita com cores, a Elysian faz silêncio. O silêncio, no contexto certo, é o som mais alto da sala.

**No mundo dark:** a cor primária é presença — o preto absoluto que abafa, sufoca, cria peso físico. Não é ausência de luz. É a presença do peso. Da máquina. Do que existe antes do humano entrar.

### 4.2 Paleta Fundacional

**Base Luminosa — Mundo Luminoso**

```
Branco Elysian:     #FFFFFF   — superfícies primárias
Off-white:          #F8F7F3   — hero luminoso, fundo principal ← IDENTIDADE DA MARCA
Névoa:              #F2F1ED   — superfícies secundárias, cards
Pedra:              #E8E7E2   — bordas sutis, divisores
```

**Base Dark — Mundo Dark**

```
Preto Absoluto:     #000000   — hero dark fase 1, impacto máximo ← IDENTIDADE DA MARCA
Preto Profundo:     #0A0A0A   — seções de tensão
Preto Técnico:      #0D0D0D   — seções de problema/dor
Fumo:               #1A1A1A   — texto principal sobre fundo claro
```

**Acento — O único momento de cor em ambos os mundos**

```
Integrity Emerald:  #10B981   — acento primário global
Emerald Hover:      #059669   — estado ativo/hover
Emerald Muted:      rgba(16,185,129,0.08) — background sutil
```

**Texto Técnico Dark:**

```
Branco Texto:       #F2F2F2   — texto principal no dark (não branco puro)
Branco Muted:       rgba(242,242,242,0.70) — corpo no dark
Branco Ghost:       rgba(242,242,242,0.40) — metadados, coordenadas
```

### 4.3 Paleta das Verticais

| Vertical | Cor | Hex | Contexto de uso |
|---|---|---|---|
| Elysian Lex | Emerald | #10B981 | Label `VERTICAL: LEX` no dark |
| CRM Aurora / Silent Operator | Índigo | #4F46E5 | Contexto de produto |
| Mad Lab Aurora | Âmbar | #D97706 | Contexto de produto |

**Regra:** No site institucional Elysian, apenas o Emerald aparece como acento. As cores de vertical aparecem somente como labels técnicos de identificação (`VERTICAL: LEX`, `STRATEGY: MAD LAB AURORA`) — nunca como cor de fundo ou elemento de UI principal.

### 4.4 Regras de Uso de Cor

**R1 — O dark não precisa de cor para ter peso.**
No mundo dark, a cor quase não existe. Preto, branco em opacidade, Emerald como sinal único. A ausência de cor *é* a cor.

**R2 — O luminoso não precisa de cor para ser rico.**
No mundo luminoso, o off-white, o preto do texto e o espaço negativo criam toda a riqueza visual necessária. O Emerald aparece como o único acento — e por ser único, tem impacto máximo.

**R3 — O acento é singular.**
Uma seção tem um momento de Emerald. Não cinco. Quando o acento está em todo lugar, ele não é acento — é cor de fundo.

**R4 — Nunca gradiente entre duas cores.**
Gradientes de cor são proibidos. Exceção única: gradiente de opacidade do mesmo tom (#000000 com 0% a 100% de opacidade) para transição entre mundos.

**R5 — O off-white tem temperatura.**
`#F8F7F3` não é "quase branco". Tem leve temperatura quente — papel premium, linho, superfície onde trabalho real acontece. Nunca substituir por `#FAFAFA` ou `#F5F5F5` — são frios demais para a identidade Elysian.

## PARTE V — SISTEMA TIPOGRÁFICO DE LOGOTIPO

### 5.1 Conceito

O logotipo Elysian é o nome como objeto — não como fonte aplicada.

No mundo luminoso: `ELYSIAN` em Söhne ExtraBold, preto sobre off-white, em escala que ocupa a tela. A palavra tem peso físico. É uma escultura tipográfica antes de ser um logo.

No mundo dark: `ELYSIAN` ou `FLYSIAN` em Söhne SemiBold, branco, em tamanho de navegação. Presente sem dominar.

**A logo muda de escala e peso entre os dois mundos — não de fonte.**

### 5.2 Variações

| Variação | Peso | Tamanho | Mundo | Uso |

## PARTE VI — SISTEMA DE IMAGEM

### 6.1 A Premissa Visual Central

Descoberta nas referências: **o humano contém o mundo dentro de si.**

Não a pessoa ao lado da tecnologia. No a tecnologia ao lado da pessoa.
A tecnologia, a cidade, o sistema, a arquitetura — existem *dentro* da silhueta humana.

Isso é a missão da Elysian em forma visual:
o humano não é substituído — ele absorve. Ele contém. Ele eleva.

### 6.2 Técnicas de Imagem Aprovadas

**Double Exposure — Técnica Principal**
A silhueta humana contém um mundo interno: cidade, arquitetura, dados, natureza.
O fundo é sempre branco ou off-white — a pessoa emerge do claro, não do escuro.
A fusão entre humano e mundo é a narrativa.

*Referências diretas: Imagens 1 e 2 do moodboard.*

**Colagem Editorial Fragmentada**
O rosto ou figura humana existe através de fragmentação — tiras verticais, cortes geométricos, deslocamentos.
Elementos técnicos ao fundo (grid, código, documentos) coexistem com o humano sem dominá-lo.
Acentos de cor cirúrgicos e únicos (não policromia).

*Referência direta: Imagem 3 do moodboard.*

**Tipografia Física com Figura**
A letra tem peso de objeto. A pessoa atravessa o texto — não fica atrás dele.
Escala extrema. A tipografia e o humano são co-protagonistas.

*Referências diretas: Imagens 4 e 5 do moodboard.*

**Fragmentação Geométrica do Humano**
O rosto fragmentado em blocos/pixels que deslocam mas não destroem a identidade.
A tecnologia transforma sem apagar. A identidade persiste através da fragmentação.

*Referência direta: Imagem 6 do moodboard.*

**Dualidade Simultânea (Dark + Luminoso numa imagem)**
A mesma figura existe no preto e no branco ao mesmo tempo.
Tipografia em negativo que recorta o rosto. Os dois mundos coexistem em uma composição.

*Referência direta: Imagem 7 do moodboard.*

### 6.3 Tratamento de Imagem

**No mundo luminoso:**
- Fundo: branco ou off-white — nunca colorido
- Saturação: reduzida 15–20% do original
- Contraste: natural, levemente aumentado para presença
- Temperatura: neutra a levemente fria
- Tratamento cromático: preto e branco ou paleta restrita de 2–3 tons

**No mundo dark:**
- Iluminação: única e dramática — uma fonte de luz (lateral, frontal baixa, contra-luz)
- Exposição: subexposta intencionalmente — o que não é revelado tem tanto peso quanto o que é
- Textura: grain de filme, ruído digital sutil — evidência de que algo real foi capturado
- Cor: quase monocromático, com variações de temperatura dentro do escuro

**Pessoas no mundo dark:**
- Estado de concentração e fluxo — nunca sorrisos
- Olhar: direto à câmera ou em perfil — nunca desviado ou casual
- O rosto emerge do escuro — não flutua em fundo neutro

### 6.4 O Que Nunca Entra

```
✗ Stock photos genéricas de "equipe diversa sorrindo"
✗ Laptops abertos como símbolo de trabalho
✗ Apertos de mão
✗ Nuvens como metáfora de "cloud"
✗ Robôs, circuits, cérebros digitais como representação de IA
✗ Fundos coloridos ou gradientes por trás de pessoas
✗ Imagens de IA geradas sem intenção artística clara
✗ Qualquer imagem que precise de legenda para fazer sentido
```

## PARTE VII — ARQUITETURA DO HERO

### 7.1 O Hero Como Declaração de Marca

O hero do site Elysian não é uma seção de entrada.
É a marca inteira em movimento.

Em 8–12 segundos, o visitante vive a missão completa:
Máquina → Humano. Tensão → Paz. Sistema → Pessoa.

### 7.2 Estrutura do Stop Motion Dual-Phase

**FASE 1 — O Mundo Dark (Tensão)**

```
Fundo:          #000000 — preto absoluto
Imagem central: Rosto humano emergindo do escuro
                Iluminação cinematográfica, única fonte de luz
                Grain de filme sutil
                O rosto não está completamente revelado

Texto técnico (background, opacidade 15–25%):
                Colunas verticais de dados descendo
                Letras, números, identificadores
                Velocidade: lenta, constante
                Evidência de sistema funcionando

Texto de UI (foreground):
                "VERTICAL: LEX" — label técnico, Emerald sutil
                "INDUSTRIAL INTELLIGENCE // AURORA_V8"
                "STRATEGY: MAD LAB AURORA"
                "ARTIFACT: 2026.Q2.CERT"
                "SVRN_OPERATOR: ACTIVE"
                Tudo em monospace capslock, tracking extremo

Navegação:
                Logo "ELYSIAN" — Söhne SemiBold, branco, esquerda
                Nav links — capslock monospace, branco 60%
                "EST. 2026 // SYRN" — direita, micro

Duração:        4–6 segundos
Transição:      Sem cut abrupto — o claro começa a emergir
```

**FASE 2 — O Mundo Luminoso (Paz)**

```
Fundo:          #F8F7F3 — off-white, temperatura quente
Imagem:         As mesmas colunas de texto técnico do dark
                Agora em #1A1A1A, opacidade 8–12%
                Presença de sistema sem dominância

Elemento central:
                "ELYSIAN" em Söhne ExtraBold
                Tamanho: ocupa ~80% da largura da tela
                Cor: #0A0A0A — quase preto
                Tracking: -4px a -6px
                O nome tem peso físico de escultura

Detalhes:
                Pequeno elemento geométrico visível dentro de uma letra
                (referência ultra-sutil aos Campos Elísios)
                Navegação em preto — os mesmos links do dark
                "EST. 2026 // SY" — direita, micro, muted

Duração:        Permanente — é o estado de repouso do hero
                O visitante começa a scrollar a partir daqui
```

### 7.3 A Transição Entre as Fases

A transição não é um corte. É uma dissolução com intenção.

```
Opção A — Dissolução de opacidade:
  Dark fade para off-white enquanto
  o texto técnico muda de cor (branco → #1A1A1A 10%)
  e o nome ELYSIAN emerge de tamanho 0 para 100%

Opção B — Reveal tipográfico:
  As letras de ELYSIAN são reveladas uma a uma
  enquanto o fundo muda de preto para off-white
  O rosto desaparece dentro das letras

Opção C — Inversão de mundo:
  A tela divide-se horizontalmente
  Dark sobe, luminoso sobe de baixo
  Os dois mundos coexistem por 1 segundo antes da fusão completa
```

**Regra:** A transição deve durar entre 1.5 e 2.5 segundos. Rápida demais = corte sem intenção. Lenta demais = o visitante perde o momentum.

### 7.4 O Que o Hero Comunica Sem Palavras

```
Dark:      "O sistema existe. É real. Tem peso. Não é marketing."
Transição: "Mas o sistema serve a algo maior."
Luminoso:  "O que é maior tem um nome. E o nome é belo."
```

O visitante não precisa ler nada para entender isso.
Ele sente antes de ler.
Isso é o que "experiência que habita a memória" significa na prática.

## PARTE VIII — ELEMENTOS DECORATIVOS E SISTEMA TÉCNICO

### 8.1 Colunas de Dados (Rain System)

Elemento visual presente em ambas as fases do hero e em seções dark.

**Características:**
- Texto em queda vertical, colunas espaçadas irregularmente
- Conteúdo: letras, números, fragmentos de palavras do universo Elysian
  (`AURORA`, `LEX`, `TRUSTWARE`, `SVRN`, `OPERATOR`, `ELYSIAN`, `AI`)
- Velocidade: lenta e constante — não Matrix, não animação de loading
- Opacidade no dark: 15–25%
- Opacidade no luminoso: 6–12%
- Fonte: monospace
- Cor no dark: branco
- Cor no luminoso: #1A1A1A

**O que não é:**
Não é referência a Matrix. Não é estética hacker. É evidência de sistema — a prova visual de que algo está funcionando por baixo.

### 8.2 Labels Técnicos

Elementos textuais que aparecem no dark como identificadores de sistema:

```
Formato:    [CHAVE]: [VALOR]
Exemplo:    VERTICAL: LEX
            ARTIFACT: 2026.Q2.CERT
            SVRN_OPERATOR: ACTIVE
            STRATEGY: MAD LAB AURORA

Fonte:      Monospace capslock
Tracking:   +0.15em a +0.25em
Opacidade:  60–80% (foreground) / 20–30% (background)
Cor:        Branco ou Emerald (#10B981) para labels de destaque
```

### 8.3 Elemento Geométrico (Referência Mitológica)

Um único elemento geométrico presente no hero luminoso — ultra-sutil.
Integrado a uma letra do nome ELYSIAN (dentro do `L` ou do `Y`).
Proporção baseada na razão áurea.
Referência visual aos portões dos Campos Elísios — sentida, não explicada.
Opacidade: 30–40% sobre o off-white.

### 8.4 Grid de Proporção (Invisível)

Todas as composições Elysian são construídas sobre um grid baseado na razão áurea (1:1.618).
Não é visível. Mas a proporção é sentida — é o que diferencia "parece certo" de "parece correto".

## PARTE X — SISTEMA DE LAYOUT E ESPAÇAMENTO

### 10.1 O Princípio do Ar

O espaço negativo na Elysian não é espaço vazio.
É onde o visitante respira, processa, e confia.

Antes de adicionar qualquer elemento a uma composição, pergunte:
*"O que eu posso remover e ainda ter a mesma comunicação?"*

### 10.2 Grid Base

```
Container máximo:    1280px
Colunas:            12
Gutters:             24px (mobile: 16px)
Margins laterais:    80px (desktop) / 24px (mobile)
Baseline grid:       8px
Razão áurea:         1.618 (grid de proporção invisível)
```

### 10.3 Sequência Narrativa de Seções

```
01. HERO           → Dark fase 1 + Luminoso fase 2 (stop motion)
02. RESPIRAÇÃO     → Luminoso. Prova social passiva. O visitante processa.
03. VALOR          → Luminoso com elementos de transição. Proposta central.
04. DOR            → Dark. A tensão antes da solução.
                     "O problema que você tem e ainda não nomeou."
05. SOLUÇÃO        → Transição dark → luminoso. O mecanismo único.
06. VERTICAIS      → Luminoso. CRM, Elysian Lex, Mad Lab Aurora.
07. PROVA          → Luminoso. Cases, evidências, dados reais.
08. CTA            → Luminoso com Emerald. Uma ação. Sem ruído.
```

## PARTE XI — CHECKLIST DE VALIDAÇÃO

```
IDENTIDADE
[ ] A fonte é Söhne ou GT America (não fallback em display)?
[ ] O tracking em display é negativo?
[ ] O texto técnico é monospace em capslock?
[ ] O peso segue a escala definida?

COR
[ ] No luminoso: fundo é off-white #F8F7F3 (não cinza genérico)?
[ ] No dark: fundo é preto absoluto #000000 (não cinza escuro)?
[ ] O acento Emerald aparece no máximo uma vez por seção?
[ ] Não há gradiente entre duas cores diferentes?

HERO
[ ] A fase dark tem rosto humano como elemento central?
[ ] O texto técnico (rain) está em opacidade correta (15–25% no dark)?
[ ] A transição entre fases dura entre 1.5 e 2.5 segundos?
[ ] No luminoso, o nome ELYSIAN tem tracking negativo e peso 800?
[ ] O elemento geométrico mitológico está presente e sutil?

IMAGEM
[ ] A imagem usa uma das técnicas aprovadas (double exposure, fragmentação, etc.)?
[ ] O humano é sempre o elemento central — nunca substituído pela tecnologia?
[ ] No dark: iluminação única e dramática?
[ ] No luminoso: fundo branco/off-white, saturação reduzida?
[ ] Não há stock photos genéricas?

DARK EDITORIAL
[ ] O dark aparece após ao menos uma seção clara de descompressão
    (exceto o hero — o hero pode começar dark)?
[ ] Os tons de preto entre seções são diferenciados?
[ ] O retorno ao luminoso é gradual (não corte abrupto)?
[ ] Passou no teste: "se remover o preto, o impacto some"?

ESPAÇO
[ ] Existe ar generoso suficiente para o conteúdo respirar?
[ ] A hierarquia de leitura é clara sem explicação?
[ ] Nenhum elemento toca a borda do container?

MITOLOGIA
[ ] A referência aos Campos Elísios é subtexto (sentida, não explicada)?
[ ] O elemento geométrico é visível mas não óbvio?
```
